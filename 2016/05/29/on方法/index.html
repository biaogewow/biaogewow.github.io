jQuery on()方法使用详解

jQuery on()方法是官方推荐的绑定事件的一个方法

$(selector).on(event,childSelector,data,function,map)

与此相似的有bind() 、live() 、delegate() ，以上三种方法在jQuery1.8之后都不推荐使用，   
官方在1.9时已经取消使用live()方法了，所以建议都使用on()方法；
on()则是整合了之前的三种方式的新事件绑定机制。

on方法的优势：

一、多个事件绑定同一个函数

$(document).ready(function(){  
    $("p").on("mouseover click",function(){  
      $(this).show();  
   });  
});

二、多个事件绑定不同函数

$(document).ready(function(){
      $("div").on({
           mouseover:function(){$("body").css("background-color","red");},  
        click:function(){$("body").css("background-color","blue");}, 
        mouseout:function(){$("body").css("background-color","yellow");}  
          });
    });

三、事件监听的委托——这是重点

事件监听的委托，利用的是事件冒泡原理，监听点击事件
可以节约性能，使用on函数有一个优点当就是动态添加新的项目进来的时候也会绑定事件
$(document).ready(function(){ 
    var current = null;
    $('ul').on('click','li',
        function (){
            $(this).css("background-color","blue"); //此时this指代当前点击的li
            current = this;
        }
    );
});
